FocusBrain — Aspect Technique

Application PWA de gestion de tâches conçue spécifiquement pour les cerveaux TDAH. Stack 100 % client-side, aucun backend, aucune base de données distante.

Vue 3 Pinia TailwindCSS Vite + PWA localStorage only 15 stores 12 vues

🎯 Logique de suggestion Dashboard
  • Énergie low → tâches faciles (≤ 30 min, non urgent/high) filtrées par byEnergy('low')
  • Énergie medium → toutes tâches pending, triées par priorité
  • Énergie high → toutes tâches, priorité aux plus difficiles
  • Fallback : première tâche de la liste pending si filtre vide
⏱️ FocusView — Timer SVG circulaire
  • Arc SVG : 282.6 px de circonférence, stroke-dasharray animé
  • Tips TDAH : tips[Math.floor(Date.now() / 300000) % tips.length] (rotation toutes les 5 min)
  • 4 dots Pomodoro visuels (cycle de 4 sessions)
  • Tâches en attente visibles en arrière-plan (blurred), cliquables pour switch

10. Patterns Architecturaux

Persistance manuelle
Pattern uniforme dans tous les stores : load() à l'init depuis localStorage, save() (JSON.stringify) après chaque mutation. Aucun plugin externe. Limites : notifications max 80 · rapports hebdo max 12 · logs XP max 50.
Gamification transversale
Le scoring XP et les unlocks d'achievements sont centralisés dans gamification.js. Les autres stores (tasks, habits, focus) appellent directement ses actions — aucune logique de scoring locale.
Focus Timer cross-navigation
Sauvegarde toutes les 10s avec un timestamp savedAt. À la restauration, le temps écoulé est recalculé. Si l'écart dépasse 2h, l'état est abandonné proprement.
Dark mode class-based
Classe dark sur <html> (Tailwind class strategy). Initialisé dans App.vue avant le premier rendu pour éviter le flash de couleur.
Commandes de développement
npm run dev      # Serveur de développement → http://localhost:5173
npm run build    # Build production → dist/
npm run preview  # Preview du build
