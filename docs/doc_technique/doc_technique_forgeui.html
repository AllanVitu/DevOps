3. Stores Pinia

Pattern commun : lecture depuis localStorage à l'init · sauvegarde JSON après chaque mutation sur cart.js.

catalog.js forge_catalog_filters

Source de vérité du catalogue produits. Charge les données depuis un fichier JSON statique et gère filtrage, tri et recherche entièrement côté client.

StateTypeValeurs
productsProduct[]Catalogue complet chargé au montage
activeFiltersobject{ category, tags[], priceMax, theme }
sortBystring'newest' · 'popular' · 'price-asc' · 'price-desc'
searchQuerystringFull-text sur name + tags
filteredProducts (computed) featuredProducts (computed) setFilter(key, val) resetFilters() setSort(val) getById(id) 
const filteredProducts = computed(() => {
  let list = products.value
  if (activeFilters.category)
    list = list.filter(p => p.category === activeFilters.category)
  if (activeFilters.tags.length)
    list = list.filter(p => activeFilters.tags.every(t => p.tags.includes(t)))
  if (searchQuery.value.trim())
    list = list.filter(p => p.name.toLowerCase().includes(searchQuery.value.toLowerCase()))
  return [...list].sort(SORT_FNS[sortBy.value])
})

4. Composants Principaux

ProductCard.vue
  • 3 variantes prop : featured · normal · compact
  • Badge rareté (Gratuit / Pro / Premium)
  • Thumbnail + tags catégorie + prix
  • Bouton "Ajouter" → cart.addItem()
  • Hover : scale + overlay "Voir le détail"
FilterBar.vue
  • Filtres multi-select : catégorie, tags, thème
  • Slider de prix max
  • Emit vers catalog.setFilter() en temps réel
  • Badge compteur de filtres actifs
  • Bouton "Tout réinitialiser" → catalog.resetFilters()
CartDrawer.vue
  • Rendu via Teleport to="body"
  • Slide-in depuis la droite (transition transform 300ms)
  • Liste items avec quantité éditable
  • Sous-total + code promo inline
  • CTA "Commander" → route #/checkout
ThemePreview.vue
  • Preview live du thème sélectionné (iframe ou inlined)
  • Toggle Dark / Light par thème
  • Bouton "Copier CSS" → Clipboard API
  • Miniature interactive des composants inclus
AppNavbar.vue
  • Logo + liens de navigation principaux
  • Badge compteur panier (cart.itemCount, réactif)
  • Icône cart → ui.openCart()
  • Toggle thème dark/light
  • Hamburger menu responsive mobile
SearchBar.vue + SortSelect.vue
  • SearchBar : input debounced 300 ms → catalog.searchQuery
  • SortSelect : 4 options de tri, recalcul computed immédiat
  • Aucune requête réseau — tout filtré côté client

7. Déploiement Railway

Pipeline de build
npm run dev      # Dev → localhost:5173
npm run build    # Vite → dist/
npm run preview  # Preview prod → localhost:4173

Railway détecte automatiquement le dist/ et sert les fichiers statiques via son proxy HTTP.

Config Railway
  • Type : Static Site
  • Build command : npm run build
  • Output directory : dist
  • HTTPS automatique (Let's Encrypt)
  • Deploy on git push (CD)
  • URL : *.up.railway.app
💡 Hash routing : le #/ de Vue Router évite tout besoin de règle de réécriture URL côté serveur. Toute la navigation se fait côté client — aucune erreur 404 sur Railway.
