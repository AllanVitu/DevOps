01 — overview

Stack & Architecture

Super Snake est une Single Page Application construite avec Vue 3 en mode Composition API. Aucun routeur externe n'est utilisé : la navigation entre les écrans est gérée via une variable réactive phase dans App.vue. Le rendu du jeu repose entièrement sur un canvas HTML5 2D animé via requestAnimationFrame.

Runtime

vue3.4.21
vite5.2.0
@vitejs/plugin-vue5.0.4

Aucune dépendance backend. Toute la persistence (skins, œufs, high score) passe par localStorage. Le jeu tourne intégralement côté client, sans réseau.

Séparation logique / rendu / son useSnake (logique pure), renderer.js (canvas), sounds.js (audio) — testabilité et clarté

04 — useSnake.js

Logique de jeu

useSnake(mode) est un composable Vue 3 qui encapsule toute la logique du jeu sans aucune référence au DOM ni au canvas. Il expose un état réactif et une API impérative appelée depuis GameCanvas.vue.

Queue de direction

Le changement de direction ne s'applique pas immédiatement — il passe par dirQueue, un buffer de 2 entrées. Cela permet au joueur de placer jusqu'à 2 virages d'avance sans perdre d'input. La direction opposée et la répétition de la même direction sont ignorées. 

function queueDir(dir) {
  const last = dirQueue.value.at(-1) ?? direction.value
  if (OPPOSITE[dir] === last || dir === last) return   // ignoré
  if (dirQueue.value.length < 2) dirQueue.value.push(dir)  // buffer max 2
}

Tick de jeu

La fonction tick(now) est appelée depuis le RAF loop de GameCanvas.vue dès que Date.now() - lastMove ≥ moveInterval. Elle :

  1. Expire les power-ups dont le timestamp until est dépassé
  2. Dépile la queue de direction
  3. Calcule la nouvelle tête en tenant compte du mode (portal / murs) et de l'inversion
  4. Vérifie la collision self (ignorée si bouclier actif)
  5. Détecte la nourriture mangée, calcule les points avec multiplicateurs et combo
  6. Déclenche l'effet du food type, fait monter le niveau tous les 10 items mangés
  7. Retourne un objet {died, ate, leveledUp} exploité par GameCanvas

Calcul du score

const mult  = (activePowerUps.multiplier && now < activePowerUps.multiplier.until) ? 2 : 1
const bonus = Math.floor(combo.value / 3)   // +1× par tranche de 3 combos
const pts   = Math.max(0, info.points) * mult * (1 + bonus)
score.value += pts

05 — renderer.js

Rendu Canvas 2D

Le module renderer.js exporte des fonctions pures qui prennent un contexte 2D en premier argument. Aucun état interne — le rendu est entièrement déterministe à partir des paramètres.

Pipeline de rendu (ordre d'appel)

drawBackground(ctx, W, H, ts)   // fond + grille + vignette
drawFoods(ctx, foods, ts)        // nourriture avec formes spécifiques
drawSnake(ctx, snake, dir, ts, powerUps, now, skin)
drawParticles(ctx, particles)   // particules burst + trail
drawFlash(ctx, W, H, color, alpha)  // overlay flash (eat/death)

drawBackground

Trois couches superposées :

  1. Fond uni #0D0018
  2. Grille en lignes violet très subtiles, opacité animée via Math.sin(time * 0.0008)
  3. Vignette radiale noir aux bords

drawFoods

Chaque food a un phase random fixé à la création pour que l'animation de pulse soit désynchronisée. Les formes sont calculées géométriquement :

FoodFormeParticularité visuelle
APPLECercleGradient radial + halo pulsant

SKULLCercle + croix yeuxFond sombre, yeux en X dessinés au stroke
RAINBOWCercleTeinte HSL qui tourne time * 0.08, ring externe

drawSnake

Le serpent est dessiné de la queue vers la tête (boucle i = length-1 → 0) pour que la tête soit toujours au-dessus. Chaque segment a un rayon qui diminue légèrement vers la queue.

La couleur des segments est interpolée linéairement entre bodyA et bodyB du skin actif selon la position dans le serpent (progress = i / length). En mode Rainbow (power-up ou skin Cosmique), la couleur est calculée en HSL tournant.

La tête (index 0) reçoit un traitement spécial : yeux directionnels, couronne animée, bouclier cyan pulsant si actif, aura spéciale ALLAN PRIME si skin.special === 'allan'.

06 — sounds.js

Audio procédural — Web Audio API

Aucun fichier audio externe. Tous les sons sont synthétisés à la volée via Web Audio API. L'AudioContext est créé en lazy singleton (premier son joué = initialisation) pour respecter la politique autoplay des navigateurs.

Fonction de base : tone()

function tone(freq, dur, vol = 0.15, type = 'sine', detune = 0) {
  const osc  = ac.createOscillator()  // générateur de fréquence
  const gain = ac.createGain()         // contrôle du volume
  osc.connect(gain); gain.connect(ac.destination)
  gain.gain.setValueAtTime(vol, ac.currentTime)
  gain.gain.exponentialRampToValueAtTime(0.001, ac.currentTime + dur)
  osc.start(); osc.stop(ac.currentTime + dur)
}

Algorithme de tirage

Le tirage se fait en deux étapes :

  1. rollRarity() — tire une rareté via une roue pondérée (total 100)
  2. rollOneSkin() — filtre les skins de cette rareté, préfère les non-possédés
function rollOneSkin() {
  const rarity  = rollRarity()
  const ofRarity = SKINS_LIST.filter(s => s.rarity === rarity && !s.alwaysOwned)
  const unowned  = ofRarity.filter(s => !ownedIds.value.includes(s.id))
  const pool     = unowned.length ? unowned : ofRarity  // préfère non-possédés
  return pool[Math.floor(Math.random() * pool.length)]
}
Pity system implicite : si tous les skins d'une rareté tirée sont déjà possédés, le pool se remplit quand même avec les skins possédés (pas de pity dur, mais préférence explicite pour les non-possédés sur chaque rareté).

GameCanvas.vue — cœur du jeu

Composant pivot qui connecte le moteur de jeu (useSnake), le rendu canvas (renderer.js), le son et les particules dans un seul RAF loop.

RAF Loop

function loop(ts) {
  // 1. Tick moteur si intervalle écoulé
  if (ts - lastMove >= moveInterval.value) { ... lastMove = ts }
  // 2. Mise à jour des particules
  updateTrail(ts); updateParticles()
  // 3. Décrémentation flash alpha
  if (flashAlpha > 0) flashAlpha = Math.max(0, flashAlpha - 0.018)
  // 4. Rendu complet
  drawBackground(ctx, W, H, ts)
  drawFoods(ctx, foods.value, ts)
  drawSnake(ctx, snake.value, direction.value, ts, activePowerUps, Date.now(), props.activeSkin)
  drawParticles(ctx, particles)
  drawFlash(ctx, W, H, flashColor, flashAlpha)
  rafId = requestAnimationFrame(loop)
}

Overlay positionné en absolute inset-0 pointer-events-none sur le canvas. Affiche score, niveau, combo (visible seulement si ≥ 3), et power-ups actifs avec countdown en secondes. Le score flash (scale-110 150ms) est déclenché via un watch(score).

11 — performance

Performance & Optimisations

Particules hors réactivité Vue

Le tableau particles dans GameCanvas.vue est un tableau JS ordinaire (const particles = []), pas une ref Vue. Cela évite que chaque ajout/suppression de particule ne déclenche un cycle de réactivité. À 60 fps avec jusqu'à plusieurs centaines de particules simultanées, cette décision est critique.

moveInterval comme computed

const moveInterval = computed(() => {
  const base = speedForLevel(level.value)
  const isSlow = activePowerUps.slow && Date.now() < activePowerUps.slow.until
  return isSlow ? Math.round(base * 1.75) : base
})

La vitesse est une valeur dérivée — jamais mutée directement, toujours recalculée. Vue invalide le cache dès que level ou activePowerUps.slow change.

Rendu canvas vs DOM

Le canvas est redessiné entièrement à chaque frame (pas de dirty-tracking). Cette approche simple fonctionne car le fond, la grille, tous les segments et les particules changent à chaque frame de toute façon. Tenter de ne redessiner que les zones modifiées serait plus complexe sans gain mesurable à cette résolution (720×480).

Nettoyage lifecycle

onUnmounted(() => {
  cancelAnimationFrame(rafId)          // arrêt du RAF
  window.removeEventListener('keydown', onKeydown)  // fuite mémoire évitée
  clearTimeout(shakeTimeout)           // timeout pending annulé
})
