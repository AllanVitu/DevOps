RégionDex Studio

Documentation complète de l'application web RégionDex Studio — un Pokédex interactif thématisant les régions françaises et européennes comme des créatures légendaires à collecter.

Vue d'ensemble

RégionDex Studio est une application web 100% frontend, sans serveur ni base de données. Toutes les données sont statiques (codées en dur dans data.js) et l'état utilisateur est persisté en localStorage. Elle est hébergeable directement sur GitHub Pages.

Constitution d'une équipe de 6 entités maximum, façon Pokémon.

Design responsive avec support du swipe, bottom sheet modal et navigation tactile.

Flux de données

Initialisation (initApp)

async function initApp() {
  // 1. Sonder les images et construire le pokedex filtré
  const pokedex = await buildPokedexFromAssets(probeImage);
  state.pokedex = pokedex;

  // 2. Charger l'état persisté (favorites, team, query…)
  hydrateStateFromStorage();

  // 3. Synchro vue depuis URL hash (#explore, #favorites, #team)
  syncViewWithHash();

  // 4. Abonner render() aux changements d'état
  subscribe(render);

  // 5. Nettoyer les IDs invalides dans favorites/team
  cleanInvalidRefs();

  // 6. Premier rendu
  render();
}

Cycle de rendu

Le rendu est complet et synchrone — à chaque changement d'état, l'intégralité du DOM visible est reconstruit.

function render() {
  const items = getDisplayList();  // Vue + filtre type + query
  renderCards(items);
  renderTypeFilters(getUniqueTypes());
  syncNavButtons();
  updateHeader(items.length);
  if (items.length === 0) renderEmptyState();
}

// Lecture du hash à l'init
function syncViewWithHash() {
  const hash = window.location.hash.replace("#", "");
  if (["explore", "favorites", "team"].includes(hash)) {
    state.view = hash;
  }
}

// Navigation programmatique
function navigateTo(view) {
  window.location.hash = view;
  state.view = view;
  notify();
}

Fonctions JavaScript (app.js)

initApp()
ASYNC

Point d'entrée de l'application. Sonde les images, hydrate l'état depuis le localStorage, s'abonne aux changements, et déclenche le premier rendu.

render()

Fonction de rendu principale. Appelée à chaque mutation d'état. Récupère la liste filtrée via getDisplayList() et met à jour l'intégralité du DOM visible.

renderCards(items: Entity[])

Vide le container .deck, puis clone et remplit le template #cardTemplate pour chaque entité. Les délais d'animation sont échelonnés (animation-delay) selon l'index de la carte.

openDetail(item: Entity)

Remplit le template #detailTemplate avec les données de l'entité, applique la couleur d'accent dynamique (--accent), et ouvre le modal en ajoutant la classe .open.

closeDetail()

Ferme le modal en retirant la classe .open. Déclenché par : bouton close, clic sur l'overlay, touche Échap.

toggleTeam(id: string)

Ajoute ou retire une entité de l'équipe. Si l'équipe est pleine (6 membres) et que l'entité n'y est pas, affiche un toast d'erreur et n'ajoute pas.

pickRandom()

Sélectionne une entité aléatoire dans le pokédex complet et ouvre sa fiche détail. Bascule sur la vue Explorer si nécessaire.

probeImage(src: string) → Promise<boolean>
ASYNC

Tente de charger une image avec un timeout de 2 secondes. Retourne true si l'image se charge, false en cas d'erreur ou de timeout. Utilisé à l'init pour filtrer les entités avec images manquantes.

showToast(message: string)

Crée et affiche un toast de notification pendant 3 secondes, puis le retire du DOM.

setupIntroScreen()

Initialise l'écran d'intro : vérifie sessionStorage.introDismissed, attache le swipe-up (seuil 50px) et le clic bouton. Le dismiss ajoute la classe .dismissed qui déclenche l'animation de sortie.

Logique de filtrage (getDisplayList)

function getDisplayList() {
  let list = state.pokedex;

  // 1. Scope de la vue
  if (state.view === "favorites")
    list = list.filter(e => state.favorites.has(e.id));
  else if (state.view === "team")
    list = list.filter(e => state.team.has(e.id));

  // 2. Filtre par type
  if (state.type !== "Tous")
    list = list.filter(e => e.types.includes(state.type));

  // 3. Recherche textuelle (nom, type, catégorie)
  if (state.query) {
    const q = state.query.toLowerCase();
    list = list.filter(e =>
      e.name.toLowerCase().includes(q) ||
      e.types.some(t => t.toLowerCase().includes(q)) ||
      e.category.toLowerCase().includes(q)
    );
  }
  return list;
}

Fonction buildPokedexFromAssets

async function buildPokedexFromAssets(probeFunction) {
  // Tester chaque image en parallèle
  const results = await Promise.all(
    allPokemon.map(p => probeFunction(p.image))
  );

  // Garder seulement les entités avec image disponible
  const available = allPokemon.filter((_, i) => results[i]);

  // Fallback : si aucune image, tout garder avec image vide
  if (available.length === 0) {
    return allPokemon.map(p => ({ ...p, image: "" }));
  }

  return available;
}

Performance

🖼️

Lazy loading

Toutes les images utilisent loading="lazy". Seules les images visibles dans le viewport sont chargées.

🔍

Sonde d'images

À l'init, les images non disponibles sont filtrées (timeout 2s) pour éviter les images cassées en production.

📌

Cache DOM

Les références DOM fréquemment utilisées sont cachées dans l'objet refs pour éviter les querySelector répétés.

📦

Zéro dépendance

Aucun framework, aucun bundler. Chargement instantané sans node_modules ni build step.

🎨

CSS natif

Animations GPU-only (transform, opacity). Transitions CSS plutôt que JS pour 60 fps garanti.

🌐

GitHub Pages ready

Fichiers 100% statiques. Déployable directement depuis le dossier docs/ sans configuration serveur.
